DNYFappbuilder Docs

Complete reference for the abp CLI — build system, device installer, signing pipeline, and deploy grid.

Installation

One-liner

curl -fsSL https://raw.githubusercontent.com/DNYFTETCH/DNYFappbuilder/main/install.sh | bash

termux-setup-storage

Manual

cd ~/dnyf-appbuilder && ./setup.sh
source ~/.bashrc

Quick Start

# 1. Create project
abp init myapp flutter

# 2. Build, sign, and install in one command
abp build myapp --target android --profile release --sign --install

# 3. Deploy to cloud
abp deploy railway

abp doctor

Scans your environment and reports which tools are installed and which are missing.

abp doctor

abp init

Scaffold a new project from a template.

abp init <name> <template>

# Examples
abp init myapp flutter
abp init api nodejs
abp init backend python

abp build

Build your application. Auto-detects project type.

abp build <path> [flags]

# Build for Android release, sign, then install
abp build . --target android --profile release --sign --install

# Build all targets in parallel
abp build . --target all --parallel --notify

abp devices

Lists all connected Android and iOS devices with model, OS version, and battery.

abp devices               # USB devices
abp devices --wireless    # WiFi ADB scan

abp install

Install an APK or IPA to connected devices.

abp install app.apk                    # Auto-select device
abp install app.apk --all              # All devices
abp install app.apk --wireless 192.168.1.10  # WiFi ADB
abp install app.apk --qr               # QR code install
abp install app.apk --verify --package com.myapp  # Verify after install

abp keygen

Generate a production release keystore using Java keytool.

abp keygen
# Output: ~/dnyf-appbuilder/config/dnyf-release.keystore

abp sign

abp sign app.apk --auto               # Auto-generates key if absent
abp sign app.apk --keystore my.jks    # Specific keystore
abp verify-sign app.apk               # Verify signature
